Começando¶

Neste projeto, voc ir avaliar o desempenho e o poder de estimativa de um modelo que foi treinado e testado em dados coletados dos imveis dos subrbios de Boston, Massachusetts. Um modelo preparado para esses dados e visto como bem ajustado pode ser ento utilizado para certas estimativas sobre um imvel em particular, seu valor monetrio. Esse modelo seria de grande valor para algum como um agente mobilirio, que poderia fazer uso dessas informaes diariamente.

O conjunto de dados para este projeto se origina do repositrio de Machine Learning da UCI. Os dados de imveis de Boston foram coletados em 1978 e cada uma das 489 entradas representa dados agregados sobre 14 atributos para imveis de vrios subrbios de Boston. Para o propsito deste projeto, os passos de pr-processamento a seguir foram feitos para esse conjunto de dados:

  • 16 observaes de dados possuem um valor 'MEDV' de 50.0. Essas observaes provavelmente contm valores ausentes ou censurados e foram removidas.
  • 1 observao de dados tem um valor 'RM' de 8.78. Essa observao pode ser considerada valor atpico (outlier) e foi removida.
  • Os atributos 'RM', 'LSTAT', 'PTRATIO', and 'MEDV' so essenciais. O resto dos atributos irrelevantes foram excludos.
  • O atributo 'MEDV' foi escalonado multiplicativamente para considerar 35 anos de inflao de mercado.

Questão 1 - Observação de Atributos¶

Para lembrar, estamos utilizando trs atributos do conjunto de dados dos imveis de Boston: 'RM', 'LSTAT' e 'PTRATIO'. Para cada observao de dados (vizinhana):

  • 'LSTAT' a porcentagem de proprietrios na vizinhana considerados de "classe baixa" (proletariado).

Implementação: Definir uma Métrica de Desempenho¶

difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. RESPOSTA DA PRIMEIRA PERGUNTA: Antes de explicar o que a grid search temos que diferenciar o que so os parmetros e os hiperparmetros de um algoritmo de aprendizagem.

Os parmetros de um modelo de aprendizagem so obtidos a partir do treinamento do modelo, por exemplo: em um modelo de regresso linear ou de regresso logstica, os parmetros estimados so os valores para cada coeficiante do modelo.

Vejamos na prtica um exemplo. Um modelo de regresso linear poderia nos fornecer o sequinte resultado (valores fictcios):

\begin{equation} MEDV = 100000 + 200000 \times RM - 40000 \times LSTAT - 5000 \times PTRATIO \end{equation}

Na regresso linear acima, os parmetros so os valores estimados para cada coeficiente do modelo. Eles so determinados a partir do prprio treinamento do modelo.

Os hiperparmetros so outra coisa: eles so parmetros que devem ser definidos ANTES que o processo de treinamento e aprendizagem do modelo comece, so uma espcie de "pressupostos" que nosso modelo utilizar para proceder com o treinamento.

Nem todos os algoritmos utilizam hiperparmetros, por exemplo: regresso linear simples no utiliza nenhum hiperparmetro, mas SVM utiliza 2 hiperparmetros principais (o Kernel e o Gamma).

Note que como os hiperparmetros devem ser definidos ANTES do aprendizado do modelo, uma tarefa importante determinar a melhor combinao de hiperparmetros para que o modelo tenha boa capacidade preditiva. Isso chamado de otimizao de hiperparmetros e o processo de escolher o melhor conjunto de hiperparmetros para o algoritmo de aprendizagem em uso.

E uma das tcnicas de otimizao de hiperparmetros a GRID SEARCH, que consiste em uma busca manual e exaustiva em um subconjunto de hiperparmetros do algoritmo de aprendizagem (uma matriz de hiperparmetros) guiada por alguma mtrica de performance (tipicamente determinada por cross-validation ou por validao em um conjunto de dados de validao separado do conjunto de dados de teste).

Um exemplo concreto ajudar a esclarecer o conceito de Grid Search: considere que vamos treinam um algoritmo de SVM. Esse algoritmo tem 2 hiperparmetros principais, o Kernel e o Gamma (existem outros hiperparmetros para a SVM, mas vamos considerar inicialmente somente esses dois). O Kernel pode ser escolhido entre as opes rbf, linear, poly, sigmoid, e o Gamma um valor numrico real positivo (para os Kernels rbf, poly ou sigmoid).

Como podemos ento estabelecer a melhor combinao entre esses hiperparmetros, sendo que o Kernel um valor nominal e o Gamme um valor real positivo? Bem, aqui usamos a GRID SEARCH e construmos uma matriz com os valores possveis do Kernel e com alguns valores escolhidos do Gamma (geralmente em escala logartmica para abranger uma grande quantidade de valores). Nossa matriz poderia se parecer com a seguinte:

Gamma \ Kernel rbf poly sigmoid
0,1
1
10
100

Agora treinaremos nosso modelo de SVM para cada combinao de Kernel/Gamma definida em nossa matriz e utilizamos alguma mtrica de avaliao (obtida, por exemplo, por cross-validation) como o F1 Score e verificamos qual combinao foi a de melhor desempenho (no exemplo fictcio abaixo, a melhor combinao foi com Kernel rfb e gamma de 100):

Gamma \ Kernel rbf poly sigmoid
0,1 F1 = 0,2 F1 = 0,4 F1 = 0,4
1 F1 = 0,3 F1 = 0,5 F1 = 0,3
10 F1 = 0,6 F1 = 0,6 F1 = 0,2
100 F1 = 0,8 F1 = 0,6 F1 = 0,2

Note que o processo da GRID SEARCH pode se tornar bastante complexo se o nmero de hiperparmetros a serem definidos maior do que 2. Com 3 hiperparmetros, teramos uma matriz tridimensional para testar (o produto cartesiano entre o conjunto de todos os valores possveis dos hiperparmetros) e com 4 hiperparmetros teramos uma matriz de quatro dimenses para testar. Quanto maior o nmero de hiperparmetros, mais complexa fica a GRID SEARCH.

RESPOSTA DA PRIMEIRA PERGUNTA: Uma das principais questes que o cientista de dados tem que responder determinar se o algoritmo de treinamento e o modelo resultante bom, ou seja, se capaz de prever bem a varivel resposta, sendo generalizvel para novos dados. Para esse treinamento e avaliao, algumas situaes so possveis:

  • Usar todos os dados para treinamento: uma possibilidade usar todos os dados para treinamento do modelo. Isso absolutamente ineficaz e errado pois simplesmente no poderemos determinar se o modelo bom. Se todos os dados foram utilizados para treinamento, como podemos saber se o modelo generalizvel? No temos mais nenhum dado para testar e esse modelo provavelmente sofrer de muito overfitting.
  • Usar um conjunto de treinamento e um conjunto de teste: dividir os dados em dois conjuntos separados (treinamento e teste) j permite que o modelo treinado seja avaliado para determinar se bom. A idia aqui treinar o modelo e estimar os parmetros com os dados de treinamento, e testar a qualidade do modelo em um conjunto de dados de teste que segue a mesma distribuio de probabilidade dos dados de treinamento (na prtica os dados so divididos aleatoriamente em dados de treinamento e de teste, garantindo assim que os dados de teste sejam representativos dos dados de treinamento). Mas essa abordagem tambm tem um problema: se quisermos otimizar os hiperparmetros dos algoritmos em uma Grid Search, por exemplo, no podemos utilizar os dados de teste, pois o conhecimento dos resultados com os dados de teste poderia influenciar a escolha dos hiperparmetros e assim gerar overfitting e diminuir a capacidade e generalizao do modelo. Os dados de teste devem ser utilizados somente no final de todo o processo, para determinar a qualidade geral do modelo: no podem ser utilizados durante o processo de treinamento em si.
  • Usar um conjunto de treinamento, um conjunto de validao e um conjunto de teste: dividindo os dados em trs conjuntos separados (mas com a mesma ditribuio de probabilidade do conjunto de treinamento) podemos treinar o modelo com o conjunto de dados de treinamento, validar os modelos e otimizar os hiperparmetros com o conjunto de dados de valiao e, aps a escolha do modelo final, testar a qualidade do modelo com o conjunto de dados de teste. Essa a melhor abordagem: usamos conjuntos de dados separados para o treinamento, validao e teste. Entretanto, essa abordagem apresenta uma desvantagem: ao separarmos o conjunto de teste dos dados, j estamos diminuindo a quantidade de dados disponveis para o treinamento do modelo; e ao separarmos mais um conjunto de dados, para a validao, estaremos diminuindo drasticamente ainda mais os dados disponveis para o treinamento e isso pode causar um efeito indesejvel: os resultados do modelo podem variar muito dependendo da escolha aleatria dos conjuntos de treinamento e validao. para resolver esse problema que utilizamos a Cross-Validation.

A Cross-Validation uma tcnica de validao para verificar como os resultados de um modelo so generalizveis para outros conjuntos de dados, ou seja, uma abordagem para avaliar a qualidade de nosso modelo, sem utilizar os dados de teste durante a validao.

De modo geral o conceito de cross-validation envolve particionar os dados de treinamento em subconjuntos complementares, realizar o treinamento do modelo em um subconjunto (que passa a ser considerado como o conjunto de treinamento), e validar o modelo obtido com o outro subconjunto (que passa a ser considerado como o conjunto de validao). Alm disso, para reduzir a variabilidade, a maioria dos mtodos de cross-validation realizada usando uma forma de "rodzio", onde mltiplas rodadas de cross-validation so realizadas e, a cada rodada, o modelo treinado e validado utilizando diferentes parties dos dados. O resultado final da validao dado por alguma forma de combinao dos resultados de cada rodada (por exemplo, a mdia) para estimar a qualidade do modelo preditivo treinado.

Em resumo, tcnicas de cross-validation combinam mtricas de avaliao do modelo para obter uma informao mais precisa a respeito da performance preditiva do modelo.

Existem diversas tcnicas para a realizao da cross-validation, tais como:

  • Cross-validation exaustiva: so mtodos que realizam o treinamento e a validao em todas as maneiras possveis de dividir os dados em conjuntos de treinamento e validao. As mais comuns so:
    • Leave-p-out cross-validation
    • Leave-one-out corss-validation
  • Cross-validation no-exaustiva: so mtodos que realizam o treinamento e a validao em algumas das maneiras possveis de dividir os dados em conjuntos de treinamento e validao. As mais comuns so:
    • K-Fold Cross-Validation
    • Holdout
    • Repeated random sub-sampling

A K-Fold Cross-Validation uma tcnica de validao cruzada no exaustiva que divide aleatoriamente o conjunto de dados de treinamento (sendo que o conjunto de dados de teste j foi separado previamente) em K grupos de mesmo tamanho. Desses K grupos, um escolhido para ser considerado como conjunto de validao, e o restante dos K - 1 grupos so utilizados no treinamento do modelo. Esse procedimento de validao repetido K vezes, cada vez utilizando um grupo diferente como dados de validao. Esses K resultados diferentes so combinados e a mdia deles utilizada como estimativa da qualidade do modelo.

O nmero exato de K grupos sujeito a debate, sendo comuns valores de 4, 5 ou 10 grupos.

Como um exemplo, vamos considerar um dataset com 100 observaes. Separamos 20% dos dados para teste, e ficamos com 80 observaes para treinamento. Dessas 80 observaes, faremos uma K-Fold Cross-Validation usando K = 4, ou seja, dividiremos as 80 observaes em 4 grupos de igual tamanho (20 observaes em cada grupo). Chamaremos esses grupos de A, B, C e D. O processo de K-Fold Cross-validation ento repedito K vezes (em nosso exemplo, 4 vezes) e, em cada repetio, um grupo diferente utilizado como validao, e o restante dos grupos (K - 1, 3 em nosso exemplo) ser utilizado para treinamento:

Rodada Grupo de Validao Grupos de Treinamento
1 A B, C, D
2 B A, C, D
3 C A, B, D
4 D A, B, C

A mdia dos K resultados das mtricas de avaliao desse modelo ser considerada como a avaliao do modelo.

RESPOSTA DA SEGUNDA PERGUNTA: A K-Fold Cross-Validation nos permite validar os modelos e otimizar os hiperparmetros atravs de algum processo de otimizao, como o Grid Search, sem que os dados de teste tenham qualquer influncia no processo de treinamento e validao. Especificamente para a Grid Search, o maior benefcio da K-Fold Cross Validation que todos os dados so utilizados para treinamento e validao (K - 1 vezes como treinamento, e 1 vez como validao), nos dando uma maior segurana que no estamos superestimando ou subestimando a qualidade do modelo (o que terima maior chance de ocorrer se estivssemos usando somente 1 grupo de validao fixo).
